无忧达 × 沃尔玛供货流程需求梳理

目标：帮助无忧达建立统一的北美海外仓面对大卖场（Walmart）的供货流程

优先级：北美仓库 → Walmart 供货全链路

一、业务全景

无忧达卖家 → 无忧达海外仓（北美）→ Walmart DC（配送中心）
                   ↕                        ↕
              我们的系统               Walmart Retail Link
         (EDI/DI/OMS/WMS)            (EDI/OTIF/Routing Guide)

核心流程：Walmart 下 PO → 无忧达仓内拣货打包贴标 → 按 Routing Guide 发货 → Walmart DC 收货验收

二、Walmart 供货合规要求

2.1 EDI 报文要求（必须）

Walmart 要求所有供应商必须通过 EDI 交互，格式为 X12 标准：

EDI 报文方向用途时效要求
850 (PO)Walmart → 供应商采购订单收到后次日 4pm CST 前确认
855 (PO Ack)供应商 → Walmart订单确认收到 PO 后 24h 内
856 (ASN)供应商 → Walmart发货通知发货当天，货到 DC 前
810 (Invoice)供应商 → Walmart发票发货后
997 (FA)双向功能确认自动

2.2 UCC-128 / GS1-128 标签要求

每个出库箱/托盘必须贴合规标签：

要素要求
SSCC-18全球唯一序列货运容器代码（箱级/托盘级）
GS1-128 条码编码 SSCC，扫描后关联 ASN
标签内容Ship From、Ship To、PO#、SSCC、件数、重量
标签位置箱体右下角短边，条码不得折叠
SSCC 与 ASN 一致性每个 SSCC 必须在 856 ASN 中出现，否则 DC 拒收

2.3 Routing Guide（运输路由指南）

要求说明
MABDMust Arrive By Date，PO 上指定的到货截止日
OTIFOn-Time In-Full，准时足量率要求 ≥98%，低于扣 3% 货款
预约发往 Walmart DC 需要提前预约送货窗口
承运商Collect freight 由 Walmart 指定承运商；Prepaid 由供应商选
托盘规格GMA 标准托盘 48×40 英寸，最高 60 英寸

2.4 OTIF 考核（核心 KPI）

  • On-Time: 货物在 MABD 当天或之前到达 Walmart DC
  • In-Full: PO 中每个 line item 数量 100% 齐全
  • 罚款: 不达标按每个 line item 货值的 3% 扣罚
  • 窗口: Collect PO 有 2 天到货窗口，Prepaid 无窗口

八、我们的系统如何让转运业务高效运转

8.1 整体价值定位

我们提供的是一站式 3PL 数字化基础设施，让海外仓不再需要拼凑多个系统，一套平台覆盖从供应商接单到商超交货的全链路：

供应商/卖家                    3PL 海外仓（无忧达）                   商超（Walmart/Costco/Target）
    │                              │                                    │
    │  ← OMS 接单+库存分配 →       │                                    │
    │                         WMS 仓内高效作业                           │
    │                         (收货→上架→拣货→打包→贴标→出库)            │
    │                              │                                    │
    │                    EDI/DI 自动对接商超系统                          │
    │                    (PO接收→确认→ASN→Invoice)                       │
    └──────────────────────────────┴────────────────────────────────────┘

8.2 四大系统如何协同

系统在转运链路中的角色核心价值
EDI商超的"数字握手通道"自动收发 Walmart/Costco 的 PO、ASN、Invoice，无需人工介入
DI数据翻译官不同格式互转（X12/JSON/XML），不同商超不同规则统一适配
OMS业务大脑接单→分仓→排程→监控，一个订单从进来到出去全生命周期管理
WMS仓内执行引擎收货、上架、拣货、包装、贴标、出库，RF 手持终端全程指导

8.3 如何帮助 3PL 仓库连接供应商和商超

上游连接（供应商/卖家）：

能力说明

订单 API卖家系统直接推单，或从电商平台自动抓单
