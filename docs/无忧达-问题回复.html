无忧达问题回复

1. Item OMS 是给卖家使用的，还是给仓库使用？

Item OMS 是一套订单履行管理系统，既可以给卖家使用，也可以给仓库使用，取决于具体业务需求。主要对接 3PL 海外仓，核心任务是把来自多个销售渠道的订单，自动路由到最合适的仓库，并跟踪从发货到签收的全过程。

2. 如果只采购 Item WMS，哪些系统与 WMS 打通？支持哪些国内 ERP？

如果只采购 Item WMS，需要通过我们的 EDI 层来打通外部系统：

  • EDI 自动对接 Walmart/Costco/Target 的 PO 并转为出库任务
  • EDI 负责数据格式转换（X12/JSON/XML）、不同商超规则统一适配

关于国内 ERP（领星 ERP、易仓 ERP 等）：EDI 支持不同格式互转，理论上可以对接任何能输出标准格式（JSON/XML/CSV）的 ERP，但具体到领星、易仓等需要评估其 API/数据格式做定制化适配。

建议：搭配 OMS + EDI 一起使用。单独 WMS 需要客户自行解决订单下发问题。

3. 沃尔玛 1P 业务（买家用 SPS Commerce），需要采购哪些系统？

必须采购的系统：EDI + OMS + WMS

订单流转流程图

SPS Commerce (买家/Walmart)
    │
    │  EDI 850 (PO 采购订单)
    ▼
EDI 系统（接收并解析 X12 报文，格式转换）
    │
    │  创建出库订单
    ▼
OMS 订单管理（库存校验 → 确认可发 → 回传 855 PO Ack）
    │
    │  生成拣货任务
    ▼
WMS 仓内执行（拣货 → 验货 → 装箱 → 贴 GS1-128 标签 → 生成 SSCC）
    │
    │  确认出库
    ▼
EDI 系统（生成 856 ASN，含 SSCC + PO 关联）
    │
    │  发送 ASN（货到 DC 前）
    ▼
SPS Commerce → Walmart DC 收货验收
    │
    ▼
EDI 发送 810 (Invoice) → 对账核销

6. Item Transload 是单独系统还是集成在 WMS 中？

Item Transload 是集成在 Item WMS 系统中的功能模块，不是独立的一套系统。

WMS 中包含转运（transload）的作业流程：收货 → 分拣 → 重新打托 → 贴标 → 出库，都在同一个 WMS 系统内完成。Transload 场景下 WMS 管理的是"货物经过仓库但不上架存储"的快速中转流程。
