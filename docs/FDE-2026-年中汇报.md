## 三、AI 在日常开发中的应用情况

### 3.1 使用的 AI 工具

| 工具 | 使用场景 | 覆盖范围 |
|------|----------|----------|
| **Kiro** | 日常编码、代码生成、重构、调试 | 核心开发工具 |
| **Cursor** | 代码编写、Copilot++ 补全 | 日常开发 |
| **Claude Code** | 复杂架构设计、代码审查、大段重写 | 架构/复杂任务 |
| **Codex (OpenAI)** | 批量代码生成、测试用例编写 | 批量任务 |
| **ChatGPT** | 需求分析、方案设计、文档撰写 | PM/设计阶段 |

### 3.2 工作方式

- **AI-Assisted Development**: 开发人员使用 AI 工具完成 60-70% 的代码编写
- **Spec-Driven**: 需求→设计→任务的结构化流程，AI 辅助拆分和生成
- **Code Review with AI**: 提交前用 AI 做初步审查，减少人工 review 负担
- **自动化报告**: AI-PM 平台自动生成日报/周报/Sprint 复盘

### 3.3 收益

- 编码效率提升约 40-60%
- 需求到代码的转化周期缩短
- 重复性工作（报告、测试用例）基本自动化
- 新人上手速度加快

### 3.4 发现的问题

- AI 生成代码需要人工校验，不能盲信
- 复杂业务逻辑 AI 容易遗漏边界情况
- 多人协作时 AI 生成代码风格需要规范统一
- 对私有 API/内部系统的理解需要额外上下文

## 四、与 Ontology / Agent Factory 的配合

### 4.1 Ontology（SAIL 部门）

- SAIL 负责 Ontology 本体管理平台开发
- 提供「Ask Ontology」AI 问答能力，集成到 Chat 频道
- AI 问答体验优化：Cypher 语句替换为自然语言描述
- 支持 Agent 辅助用户 Publish 本体

### 4.2 Agent Factory（DTS + AI Agent 部门）

- **DTS** 主导 Agent Factory 核心平台开发：
  - Multi-Agent RUN 支持
  - 会话恢复
  - IDE Builder 创建 Agent 方案
  - Claude 模型集成与速度优化
  - Marketplace 管理（统计、卡片、同步）
  
- **AI Agent** 负责：
  - Factory Runtime 运行时优化
  - Fix Agent 分析能力
  - 上线部署与联调

### 4.3 协作模式

- DTS 提供 Factory 平台能力 → AI Agent 负责上层应用落地
- SAIL 提供 Ontology 知识底座 → 通过 Chat + Agent 对外输出
