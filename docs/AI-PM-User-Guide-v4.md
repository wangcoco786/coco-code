## 平台简介

AI-PM Platform 是一款面向研发团队的智能项目管理平台，核心特性：

- **Jira 深度集成**：实时同步 Sprint、Issue、人员数据
- **AI 智能分析**：每个模块内置 AI 分析，自动识别风险和优化建议
- **五维绩效评估**：基于 SPACE + DORA 模型的客观绩效打分
- **企微推送**：风险预警、报告摘要一键推送企业微信群
- **多语言支持**：中文 / English / 日本語 / Español
- **角色差异化**：PM 和 DEV 角色看到不同的功能模块

### 技术架构

| 层 | 技术 |
|----|------|
| 前端 | React 18 + TypeScript + Vite |
| 状态管理 | TanStack Query（自动缓存） |
| 后端 | Express（Jira API 代理 + SSO） |
| AI 引擎 | 大模型 API 集成 |
| 通知 | 企业微信 Webhook |
| 部署 | 内网 Node.js + 公网 Vercel |

## Dashboard（项目驾驶舱）

选择项目后进入 Dashboard，包含 5 个子标签：

### 全局视图（PM 默认）

- **Sprint 概览**：Sprint 名称、日期范围、总任务数
- **完成率仪表盘**：环形图展示当前完成百分比
- **任务状态分布**：饼图（待办/进行中/评审/测试/完成）
- **团队负载**：按人员展示任务数量分布
- **燃尽图**：理想线 vs 实际线对比
- **风险摘要**：高/中/低风险数量速览
- **未分配任务提醒**：自动高亮未分配的高优任务

### 个人视图（DEV 默认）

- **我的任务列表**：当前迭代中分配给自己的任务
- **个人进度统计**：完成/进行中/待办数量
- **待处理事项**：需要关注的阻塞或超期任务

### 部门绩效

详见 [部门绩效](#部门绩效) 章节。

### Release Notes

详见 [Release Notes](#release-notes) 章节。

### AI 决策（仅 PM）

- 综合完成率、风险数量、任务状态的 AI 建议
- 高危风险清单及推荐行动方案

### AI 洞察

每个视图顶部有 **AI 分析** 按钮，点击后：
- 自动汇总当前数据
- 生成健康度评估
- 输出关键风险提示

## Sprint 管理

从左侧菜单进入「Sprint 管理」，包含 4 个 Tab：

### 执行看板（Board）

5 列看板展示：待办 → 进行中 → 评审 → 测试 → 完成

- **筛选**：按优先级、负责人、关键词过滤
- **任务卡片**：显示 ID（可点击跳转 Jira）、标题、优先级徽章、负责人头像
- **我的任务高亮**：当前用户的任务有特殊标记
- **CFD 图表**：累积流量图展示各状态任务变化趋势
- **Sprint 完成预测**：AI 预测是否能按时完成，不达标时自动预警

### 资源视图（Resource）

- **团队概要栏**：总任务数、已分配、未分配、平均负载、超载/均衡/空闲人数
- **开发者画像卡片**：
  - 每人一张卡片，显示名字、角色、负载百分比
  - 负载条：红色（超载 >100%）、绿色（均衡）、橙色（空闲 0%）
  - 任务列表：只展示该人作为 **Developer** 的任务（与绩效一致）
  - 任务按状态排序（进行中 > 待办 > 完成），可折叠
- **排序与筛选**：按负载、任务数、姓名排序；按超载/均衡/空闲过滤
- **团队热力图**：按时间范围展示团队工作强度

### 变更检测（Change）

- **变更类型统计**：优先级变更、新增需求、范围蔓延、状态回退
- **影响分析**：高/中/低影响分级
- **范围蔓延预警**：新增需求超过基线 20% 时自动警告
- **AI 变更分析**：一键让 AI 分析变更原因和影响

### AI 规划（Plan）

- 从 Backlog 获取候选任务
- AI 对每个任务评分（优先级 × 复杂度 × 依赖关系）
- 一键选择 Top 10 推荐任务
- 复制计划到剪贴板

## 风险与协作

> 仅 PM 角色可见

### 风险看板

按 4 个阶段分列：已识别 → 评估中 → 应对中 → 已关闭

风险类型：
- **未分配高优**：P0/P1 任务没有负责人
- **超工时**：实际工时超出预估
- **长期无更新**：任务超过 N 天未更新状态

每张风险卡片显示：风险等级、描述、关联负责人、一键推送按钮。

### 跨团队协作

- 自动识别跨项目组的协作任务
- 展示协作双方、共同任务数、完成进度

### 依赖管理

- 展示跨团队依赖关系图
- 标记阻塞中的依赖

### 企微推送

一键将高风险推送到企业微信群通知。

## 部门绩效

### 评估模型

基于 **SPACE + DORA** 框架的五维度绩效评估：

| 维度 | 权重 | 计算方式 |
|------|------|----------|
| 吞吐量 | 25% | 完成任务数 × 复杂度因子，团队内百分位排名 |
| 效率 | 30% | 平均 Cycle Time + Sprint 内按时交付率 |
| 质量 | 25% | 返工率（Reopen 次数）+ Bug 引入率 |
| 影响力 | 10% | 高优先级任务完成比 + 阻塞解决速度 |
| 协作 | 10% | 跨团队评论数 + 他人任务参与度 |

### 绩效等级

| 分数区间 | 等级 | 颜色 |
|----------|------|------|
| 80-100 | Excellent（优秀） | 🟢 绿色 |
| 60-79 | Good（良好） | 🔵 蓝色 |
| 40-59 | Average（一般） | 🟠 橙色 |
| 0-39 | Needs Improvement（需改进） | 🔴 红色 |

### 视图层级

1. **部门总览**：各维度平均分、雷达图、总完成任务数、平均 Cycle Time
2. **个人绩效**：
   - 按角色分组（Developer / Reporter / Others）
   - 每人展示五维雷达图、综合分数、等级
   - **任务列表**：展示该成员作为 Developer 的任务（与资源视图一致）
3. **绩效趋势**：折线图展示历史 Sprint 综合绩效变化

### Sprint 选择

- 默认显示当前活跃 Sprint 数据
- 下拉可选择历史已关闭 Sprint 查看绩效
- 支持与上一迭代对比

### 部门排行榜

未选项目时展示所有部门/项目的绩效排行：
- 综合分数排名
- 各维度得分柱状图
- 点击卡片进入该项目详情

### 任务归属规则

绩效中每个任务**只归属一个人**，优先级：
1. **Developer 字段**（Jira 自定义字段 `customfield_11000` 或 `customfield_11103`）
2. 没有 Developer 时使用 **Assignee**
3. 没有 Assignee 时使用 **Reporter**

此规则与资源视图完全一致，确保同一个人在两个视图中看到的任务列表相同。

## 自动化规则

> 仅 PM 角色可见

### 概念

自动化规则由「条件」+「动作」组成：当满足某个条件时，自动执行指定动作。

### 条件类型

| 条件 | 说明 |
|------|------|
| status_change | 任务状态变更 |
| priority_change | 优先级变更 |
| timeout_no_update | 超过 N 天无更新 |
| assignee_change | 负责人变更 |
| scope_change | 范围变更（标签/Story Points） |

### 动作类型

| 动作 | 说明 |
|------|------|
| send_wecom | 推送企业微信通知 |
| change_status | 自动变更状态 |
| assign_member | 自动分配负责人 |
| add_label | 添加标签 |
| generate_report | 生成报告 |

### 预设模板

- 高危风险自动通知（P0 进入开发 → 推送企微）
- 超时任务提醒（3 天无更新 → 推送企微）
- 每日进度汇总（状态变更 → 生成报告）

## 通用功能

### 全局搜索（Ctrl+K）

- 搜索页面名称快速导航
- 搜索任务 ID 或标题
- 键盘方向键选择，Enter 跳转

### Jira 跳转

所有任务 ID（如 RMS-2886）均可点击直接跳转到 Jira 对应页面。

### 多语言切换

顶部导航栏切换语言：🇨🇳 中文 / 🇺🇸 English / 🇯🇵 日本語 / 🇪🇸 Español

### 角色切换

顶部导航栏切换 PM / DEV 角色，不同角色看到的功能模块和数据范围不同。

### 项目组支持

支持将多个 Jira 项目合并为一个虚拟项目组：
- **IDC**：RP + TRF + APS
- **EAG**：BP + CRMC + VRM + OW + RE

选择项目组时，数据自动合并展示。

### Sprint 分组

同一个 Jira 项目中如有多个并行 Sprint（如 DTS 项目的 OB/OMS/DI/CP），系统自动按前缀分组展示。

## FAQ

### Q: 数据多久同步一次？
A: 页面加载时自动从 Jira 获取最新数据，TanStack Query 缓存 5 分钟。也可手动点击「同步」按钮强制刷新。

### Q: 绩效中某个人的任务列表和资源视图不一致？
A: 系统统一按 Developer 字段归属任务。如果 Jira 中某个 Issue 的 Developer 是 A，Assignee 是 B，该任务在两个视图中都归属 A。

### Q: 企微推送没收到？
A: 检查「设置 → Jira 集成」中连接状态是否正常，确认环境变量中配置了正确的企微 Webhook 地址。

### Q: 如何排除已离职人员？
A: 进入「设置 → 排除人员」，输入与 Jira 显示一致的姓名即可。排除后该人员不会出现在资源视图和绩效中。

### Q: 支持哪些浏览器？
A: 推荐 Chrome / Edge 最新版。Safari 和 Firefox 也支持。
