AI-PM 绩效评估指南

五维度评估模型 · 如何提升你的绩效分

📊 综合绩效公式

绩效分 = 吞吐量×20% + 效率×25% + 质量×25% + 影响力×15% + 协作×15%
🟢 80-100 Excellent 🔵 60-79 Good 🟠 40-59 Average 🔴 0-39 Needs Improvement

1️⃣ 吞吐量（20%）— 你完成了多少

计算方式

统计 Sprint 内完成的任务数 × 复杂度因子，在团队中算百分位排名。

复杂度加分条件加分说明
子任务 >3+0.5任务拆分多
子任务 >6+1.0大型任务
关联 issue >2+0.3跨模块
关联 issue >5+0.6高度关联
评论 >5 条+0.3有讨论
评论 >10 条+0.6复杂讨论
跨 Sprint+0.3/次持续大任务

💡 如何提升

  • 在 Sprint 内尽量多完成 Jira ticket
  • 接复杂任务（多子任务/多关联），权重更高
  • 不要只做简单低复杂度任务
  • 确保任务 status 移到 Done（只有 Done 才算完成）

2️⃣ 效率（25%）— 你完成得多快

计算方式

Cycle Time（In Progress → Done 天数）+ Sprint 内按时完成比例，各占 50%。

Cycle Time 分量 = max(0, 100 - 平均天数×5)。意味着 20 天以上=0 分，4 天=80 分，0 天=100 分。

💡 如何提升

  • 及时更新 Jira 状态：开始做就移到 In Progress，做完立刻移到 Done
  • 减少等待时间（及时找人 review，不要卡着）
  • 小批量交付，不要积攒到 Sprint 最后一天
  • 拆分大任务（20 天的任务 Cycle Time 得分为 0）
  • Sprint 结束前完成承诺的任务（Delivery Rate）

3️⃣ 质量（25%）— 你做得多好

计算方式

返工率（被 Reopen 次数）+ Bug 引入率（你完成的任务关联了 Bug 的比例）

返工率 0 + Bug 引入率 0 = 质量满分 100

💡 如何提升

  • 一次做对：减少被 Reopen 的次数
  • 开发前理解清楚需求，有疑问先确认
  • 自测充分再提交（覆盖边界情况）
  • Code Review 前自查代码
  • 写单元测试，减少 Bug 引入

4️⃣ 影响力（15%）— 你做的事有多重要

计算方式

高优先级（P0/P1）任务完成占比 × 60% + 阻塞解决速度 × 40%

💡 如何提升

  • 主动接 P0/P1 任务：高优任务占总分的 60%
  • 快速解决阻塞性问题（如果你的任务 block 了别人）
  • 不要只挑 P3 简单任务
  • 帮团队解决卡点和紧急问题

5️⃣ 协作（15%）— 你帮助团队多少

计算方式

跨团队评论数（在别人任务上发评论）+ 你参与了多少别人的任务

10 条跨团队评论即可评论分量满分

💡 如何提升（最容易提分的维度）

  • 在别人的 Jira ticket 上留评论（Code Review、建议、问题讨论）
  • 10 条跨团队评论 = 评论分量满分
  • 参与更多不同同事的任务（覆盖面比深度重要）
  • 积极参与 Code Review
  • 帮助同事排查问题并留下记录

🚀 快速提分行动清单

优先级行动影响维度预期提升
⭐⭐⭐及时更新 Jira 状态（In Progress/Done）效率+20~30
⭐⭐⭐Sprint 内完成承诺的任务效率+吞吐量+15~25

❓ 常见问题

Q: 为什么我完成了很多任务但吞吐量分不高？
吞吐量是团队内相对排名。如果别人完成得更多，你的百分位就低。另外简单任务复杂度因子只有 1.0，大任务可以到 5.0。
Q: 我的效率分为什么是 0？
检查 Jira 状态更新。如果你没有把任务移到 In Progress 就直接 Done，系统无法计算 Cycle Time，会用 createdAt → updatedAt 作为 fallback（通常很长）。
Q: 怎样让质量满分？
不被 Reopen + 你完成的任务没有关联 Bug = 100 分。
Q: 协作分怎么快速提升？
最简单：在 10 个不同同事的 Jira ticket 上各留一条有价值的评论。
Q: 为什么影响力分很低？
如果你只做 P2/P3 任务，高优完成比为 0，影响力最多 20 分。主动接 P0/P1 任务是关键。
