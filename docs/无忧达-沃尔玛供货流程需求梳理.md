# 无忧达 × 沃尔玛供货流程需求梳理

> 目标：帮助无忧达建立统一的北美海外仓面对大卖场（Walmart）的供货流程
> 优先级：北美仓库 → Walmart 供货全链路

---

## 一、业务全景

```
无忧达卖家 → 无忧达海外仓（北美）→ Walmart DC（配送中心）
                   ↕                        ↕
              我们的系统               Walmart Retail Link
         (EDI/DI/OMS/WMS)            (EDI/OTIF/Routing Guide)
```

**核心流程**：Walmart 下 PO → 无忧达仓内拣货打包贴标 → 按 Routing Guide 发货 → Walmart DC 收货验收

---

## 二、Walmart 供货合规要求

### 2.1 EDI 报文要求（必须）

Walmart 要求所有供应商必须通过 EDI 交互，格式为 X12 标准：

| EDI 报文 | 方向 | 用途 | 时效要求 |
|---------|------|------|---------|
| **850** (PO) | Walmart → 供应商 | 采购订单 | 收到后次日 4pm CST 前确认 |
| **855** (PO Ack) | 供应商 → Walmart | 订单确认 | 收到 PO 后 24h 内 |
| **856** (ASN) | 供应商 → Walmart | 发货通知 | 发货当天，货到 DC 前 |
| **810** (Invoice) | 供应商 → Walmart | 发票 | 发货后 |
| **997** (FA) | 双向 | 功能确认 | 自动 |

### 2.2 UCC-128 / GS1-128 标签要求

每个出库箱/托盘必须贴合规标签：

| 要素 | 要求 |
|------|------|
| **SSCC-18** | 全球唯一序列货运容器代码（箱级/托盘级） |
| **GS1-128 条码** | 编码 SSCC，扫描后关联 ASN |
| **标签内容** | Ship From、Ship To、PO#、SSCC、件数、重量 |
| **标签位置** | 箱体右下角短边，条码不得折叠 |
| **SSCC 与 ASN 一致性** | 每个 SSCC 必须在 856 ASN 中出现，否则 DC 拒收 |

### 2.3 Routing Guide（运输路由指南）

| 要求 | 说明 |
|------|------|
| **MABD** | Must Arrive By Date，PO 上指定的到货截止日 |
| **OTIF** | On-Time In-Full，准时足量率要求 ≥98%，低于扣 3% 货款 |
| **预约** | 发往 Walmart DC 需要提前预约送货窗口 |
| **承运商** | Collect freight 由 Walmart 指定承运商；Prepaid 由供应商选 |
| **托盘规格** | GMA 标准托盘 48×40 英寸，最高 60 英寸 |

### 2.4 OTIF 考核（核心 KPI）

- **On-Time**: 货物在 MABD 当天或之前到达 Walmart DC
- **In-Full**: PO 中每个 line item 数量 100% 齐全
- **罚款**: 不达标按每个 line item 货值的 3% 扣罚
- **窗口**: Collect PO 有 2 天到货窗口，Prepaid 无窗口

## 三、端到端业务流程

```
┌─────────────────────────────────────────────────────────────────────────┐
│ 1. 订单接收                                                              │
│    Walmart EDI 850 (PO) → EDI 系统解析 → OMS 创建出库订单               │
└─────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────┐
│ 2. 订单确认                                                              │
│    OMS 校验库存 → 确认可发 → EDI 回传 855 (PO Acknowledgment)           │
└─────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────┐
│ 3. 仓内作业                                                              │
│    WMS 生成拣货任务 → 拣货 → 验货 → 装箱 → 贴 GS1-128/UCC 标签         │
│    → 生成 SSCC → 关联 PO Line Item → 打托/称重                          │
└─────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────┐
│ 4. 发货 & ASN                                                            │
│    WMS 确认出库 → EDI 生成 856 (ASN) → 包含 SSCC + PO 关联              │
│    → 按 Routing Guide 预约 DC 窗口 → 交承运商                           │
└─────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────┐
│ 5. 对账 & 发票                                                           │
│    确认 DC 收货 → EDI 发送 810 (Invoice) → 对账核销                     │
└─────────────────────────────────────────────────────────────────────────┘
```

## 四、我们的系统能力对照

### 4.1 系统能力矩阵

| 环节 | Walmart 要求 | 我们的系统 | 能力状态 |
|------|-------------|-----------|---------|
| PO 接收 | EDI 850 X12 格式 | **EDI** | ✅ 可解析 X12 报文 |
| PO 确认 | 24h 内回传 855 | **EDI + OMS** | ✅ 自动生成 855 |
| 库存校验 | 确认可供数量 | **OMS + WMS** | ✅ 实时库存查询 |
| 拣货装箱 | 按 PO Line Item 拣货 | **WMS** | ✅ 波次拣货 |
| UCC/GS1 标签 | SSCC-18 + GS1-128 条码 | **WMS** | ⚠️ 需确认标签模板是否合规 |
| SSCC 生成 | 全球唯一编码 | **WMS** | ⚠️ 需配置 GS1 Company Prefix |
| ASN 发送 | 856 含 SSCC 关联 | **EDI + WMS** | ✅ WMS 出库 → EDI 生成 856 |
| Routing Guide | MABD 管控 + DC 预约 | **OMS** | ⚠️ 需新增 MABD 倒排逻辑 |
| OTIF 监控 | 准时率 + 足量率 | **OMS/DI** | ❌ 需开发 OTIF 看板 |
| Invoice | EDI 810 | **EDI** | ✅ 可生成 810 |

### 4.2 各系统职责分工

| 系统 | 在 Walmart 流程中的职责 |
|------|----------------------|
| **EDI** | X12 报文收发（850/855/856/810/997）、格式转换、通信管道 |
| **DI** (Data Integration) | 数据映射转换、多格式适配、报文校验 |
| **OMS** | 订单生命周期管理、库存分配、MABD 倒排、路由分仓、OTIF 监控 |
| **WMS** | 仓内作业执行、拣货/装箱/贴标、SSCC 生成、出库确认 |

## 七、风险关注

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| SSCC 不一致导致 DC 拒收 | 退货 + 扣罚 | WMS 出库时强制校验 SSCC ↔ ASN |
| ASN 晚于货物到达 | OTIF 不达标 | 出库即发 ASN，不等物流确认 |
| 标签不合规（位置/内容） | 扣罚 + 退货 | 首批打样 Walmart 合规部确认 |
| MABD 管控缺失 | 大面积迟到 | OMS MABD 倒排 + T-3 预警 |
| GS1 Company Prefix 未注册 | 无法生成合规 SSCC | 提前帮客户申请 GS1 |

## 八、我们的系统如何让转运业务高效运转

### 8.1 整体价值定位

我们提供的是**一站式 3PL 数字化基础设施**，让海外仓不再需要拼凑多个系统，一套平台覆盖从供应商接单到商超交货的全链路：

```
供应商/卖家                    3PL 海外仓（无忧达）                   商超（Walmart/Costco/Target）
    │                              │                                    │
    │  ← OMS 接单+库存分配 →       │                                    │
    │                         WMS 仓内高效作业                           │
    │                         (收货→上架→拣货→打包→贴标→出库)            │
    │                              │                                    │
    │                    EDI/DI 自动对接商超系统                          │
    │                    (PO接收→确认→ASN→Invoice)                       │
    └──────────────────────────────┴────────────────────────────────────┘
```

### 8.2 四大系统如何协同

| 系统 | 在转运链路中的角色 | 核心价值 |
|------|------------------|---------|
| **EDI** | 商超的"数字握手通道" | 自动收发 Walmart/Costco 的 PO、ASN、Invoice，无需人工介入 |
| **DI** | 数据翻译官 | 不同格式互转（X12/JSON/XML），不同商超不同规则统一适配 |
| **OMS** | 业务大脑 | 接单→分仓→排程→监控，一个订单从进来到出去全生命周期管理 |
| **WMS** | 仓内执行引擎 | 收货、上架、拣货、包装、贴标、出库，RF 手持终端全程指导 |

### 8.3 如何帮助 3PL 仓库连接供应商和商超

**上游连接（供应商/卖家）：**

| 能力 | 说明 |
|------|------|
| 多卖家管理 | 一个仓同时服务多个卖家，库存/账单/权限完全隔离 |
| 入库预约 | 卖家线上预约送货时段，仓库提前安排月台和人力 |
| 库存可视化 | 卖家实时查看自己在仓内的库存、效期、批次 |
| 订单 API | 卖家系统直接推单，或从电商平台自动抓单 |

**下游连接（商超/零售商）：**

| 能力 | 说明 |
|------|------|
| EDI 自动对接 | Walmart/Costco/Target 的 PO 自动接收并转为出库任务 |
| 合规标签 | 按商超要求自动生成 GS1-128/UCC 标签（SSCC 唯一编码） |
| ASN 自动回传 | 出库即发 856 ASN，DC 收货前数据先到 |
| 多商超路由 | 同一 SKU 可同时供 Walmart、Costco、Amazon，OMS 按规则分配 |
| OTIF 管控 | MABD 倒排排程，防止迟到；装箱校验，防止短装 |

### 8.4 转运效率提升点

| 环节 | 传统方式 | 用我们系统后 | 效率提升 |
|------|---------|------------|---------|
| PO 接收 | 人工看邮件/传真，手工录入 | EDI 自动解析，秒级入系统 | 从 2h → 0min |
| 订单确认 | 人工查库存，打电话确认 | OMS 自动校验库存，自动回传 855 | 从 4h → 即时 |
| 拣货 | 纸单拣货，容易错 | WMS RF 引导，路径优化，准确率 99.9% | 效率 +40% |
| 贴标 | 手工打标签，容易贴错箱 | WMS 自动生成 SSCC，扫描验证一致性 | 错误率 → 0 |
| ASN 发送 | 人工做 Excel 再转格式 | 出库自动触发 856，零人工 | 从 30min → 0min |
| OTIF 追踪 | 事后才知道被罚 | 实时看板，T-3 预警 | 罚款减少 80%+ |
