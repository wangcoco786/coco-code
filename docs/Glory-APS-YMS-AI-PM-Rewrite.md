## Slide 1: Glory 客户 — 全球供应链与系统重构 (Henan Factory)

### 业务背景

河南工厂作为全球皮料供应商，涉及原材料粗加工与成品生产。材料采用双单位管理（平方英尺/公斤 + 序列号追踪），成品带有全球唯一 UPC 码。

**核心诉求：** 支持复杂的跨国供应链路由（直发、规避关税的中转、海外前置仓），实现 AX/EDI/OMS/WMS 核心链路重构。

---

### 核心履约场景与系统规则

**四种路由场景（OMS 自动化分配）：**

| # | 场景 | 说明 |
|---|------|------|
| 1 | 直发客户 (Direct) | 如 Costco 订单直发 |
| 2 | 关税规避中转 | 成品运至泰国深加工，以"Made in Thailand"名义出口至美国 |
| 3 | 跨国前置仓 | 发往欧洲、日本等地备货 |
| 4 | 纯中转 | 中转工厂处理流程 |

**异构系统协同：**
- 美国仓使用标准 WMS
- 泰国/河南工厂使用 Vendor Portal（简化进销存）

**双单位/SN 管理：** 材料出库需换算（如 18.5 尺需匹配多张皮料 SN），成品走 UPC 管理。

### AI-PM 转型后的角色定位（对外交付 PM）

| 维度 | 传统 PM 做法 | AI-PM 转型后做法 |
|------|-------------|-----------------|
| 业务理解 | 等开发团队翻译需求 | **通过 Ontology + AI 直接读取供应链数据模型**，快速理解 OMS 路由、双单位换算等业务逻辑 |
| 需求对接 | 纯需求记录转发 | **直接参与客户会议**，拉通美国运营、河南/泰国工厂，梳理多段路由的实操细节与合规要求 |

| 异常闭环 | 人工设计测试场景 | **AI 辅助识别边缘场景**（内调单拦截、Partial Shipment、多 SN 匹配），自动生成测试用例 |

**核心价值：** PM 不再是"传声筒"，而是能直接理解跨国供应链业务、输出可运行 Demo 的技术型交付 PM。

## Slide 3: APS 核心逻辑架构（七层架构）

### 七层架构设计

| 层级 | 名称 | 核心内容 |
|------|------|---------|
| 1 | 基础数据层 (Data Foundation) | HRM/IAM（人员/技能/账号权限）、WMS（订单/任务数据）、标准作业产能（SOP/UPH） |
| 2 | 过滤池 (Filtering) | 拦截缺少技能标签人员、排除不完整订单与无标准产能记录的作业节点 |
| 3 | 数据处理层 (Processing) | 基于时间切片(Time-sliced)聚合实时需求产能与出勤算力，动态计算匹配缺口 |
| 4 | 算法核心引擎 (Algorithm) | 多约束目标求解（工时合规、成本最优、订单不逾期）+ 动态插单重算规则 |
| 5 | 预测与排班 (Forecasting) | Headcount 预测输出，跨仓位劳动力调度（借调建议） |
| 6 | 执行下发 (Execution) | 生成具体到人、时段的工作指令，同步至 WMS RF 手持终端 |
| 7 | 归档与报表 (Archiving) | 实际产能 vs 预测产能差异分析，模型自迭代与运营指标大屏 |

## Slide 4: LinkW 业务背景与 YMS 园区管理

### LinkW 业务背景与挑战

**小品牌跨境物流痛点：** 无法整箱发货，只能依靠托盘或小件零散运输，物流成本高且效率低。

**LinkW 平台解决方案：**
- **订单整合**：整合国内小商户订单，集中订舱发货至美国仓库
- **运输与仓储**：下游对接 Amazon、Costco 等零售商，承担运输仓储功能
- **Dropship（一件代发）**：减少店家库存压力，避免库存积压

**全局路由：**
```
LinkW (3PL) 国内订单集拼 → 美国转运中心 → Amazon (FBA) / Costco (Dropship)
```

### 未来计划与自动化目标

| 方向 | 具体内容 |
|------|---------|
| 仓库自动化 | 接入传送带等自动化设备，实现货物快速流转；减少人工干预 |
| 园区智能调度 | 引入视觉摄像头 + 后台远程监控；最终实现高度自动化的 YMS |

## Slide 5: Link W (YMS) vs WMS 系统功能对比

### 功能对比

| 功能模块 | WMS 现有能力 | YMS 核心价值 |
|----------|------------|------------|
| **预约管理 (Appt)** | ✓ 支持入库/出库预约，关联 PO/SO 和承运商 | ✗ 不包含业务单据层面的时段规划 |
| **月台管理 (Dock)** | 部分支持（基础指派，缺乏动态排队逻辑） | ✓ 基于任务动态指派、冲突检测与实时调度 |
| **入场签到 (Check-In)** | 人工操作，登记基础信息 | ✓ 自动化登记（车牌、封签、温度记录等），分配排队号 |
| **场区调度 (Shuttle)** | 不支持 | ✓ 管理穿梭车任务（等待区→月台），监控车辆滞留 |

### YMS 业务流程落地关键步骤

1. 司机到达道闸，使用 YMS 完成 Check-in，记录车辆状态并生成排队号
2. YMS 根据当前月台(Dock)占用情况，下发驶入指令或分配至 Buffer 停车区
3. 月台释放后，YMS 触发 Shuttle 任务，引导车辆靠泊对应月台
4. WMS 接管实物作业（装卸货），完成后通知 YMS
5. 车辆驶离月台，YMS 完成 Check-out 闭环记录
