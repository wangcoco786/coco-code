# AI-PM 绩效评估指南

## 综合绩效公式

```
绩效分 = 吞吐量×20% + 效率×25% + 质量×25% + 影响力×15% + 协作×15%
```

| 等级 | 分数 | 含义 |
|------|------|------|
| 🟢 Excellent | 80-100 | 表现突出 |
| 🔵 Good | 60-79 | 达标良好 |
| 🟠 Average | 40-59 | 有提升空间 |
| 🔴 Needs Improvement | 0-39 | 需要关注 |

---

## 1. 吞吐量（20%）— 你完成了多少

### 怎么算的
- 统计你在 Sprint 内完成的任务数
- 每个任务乘以复杂度因子（1.0~5.0）得到加权完成量
- 与团队其他人比较，算百分位排名

### 复杂度因子加分项
| 条件 | 加分 | 说明 |
|------|------|------|
| 子任务 >3 个 | +0.5 | 任务拆分多说明复杂 |
| 子任务 >6 个 | +1.0 | 大型任务 |
| 关联 issue >2 | +0.3 | 涉及跨模块 |
| 关联 issue >5 | +0.6 | 高度关联 |
| 评论 >5 条 | +0.3 | 讨论多说明有挑战 |
| 评论 >10 条 | +0.6 | 复杂讨论 |
| 跨 Sprint | +0.3/次 | 持续性大任务 |

### 💡 如何提升
- **多完成任务**：在 Sprint 内尽量多完成 Jira ticket
- **接复杂任务**：有子任务、关联多的任务权重更高
- **不要只做简单任务**：复杂度因子让大任务更有价值
- **避免半途而废**：只有 status=Done 的才算完成

---

## 2. 效率（25%）— 你完成得多快

### 怎么算的
```
效率 = (CycleTime分量 + DeliveryRate分量) / 2
CycleTime分量 = max(0, 100 - 平均天数×5)
DeliveryRate分量 = Sprint结束前完成数 / 总分配数 × 100
```

- **Cycle Time**：从 In Progress 到 Done 的天数，越短越好
- **Delivery Rate**：Sprint 结束前按时完成的比例

### 💡 如何提升
- **及时更新 Jira 状态**：开始做就移到 In Progress，做完立刻移到 Done
- **缩短 Cycle Time**：
  - 拿到任务后尽快开始（不要放着不动）
  - 减少等待时间（及时找人 review）
  - 小批量交付，不要积攒到最后
- **Sprint 内完成承诺的任务**：不要拖到下个 Sprint
- **20 天以上的任务 Cycle Time 得分为 0**：拆分大任务

---

## 3. 质量（25%）— 你做得多好

### 怎么算的
```
质量 = ((1-返工率)×100 + (1-Bug引入率)×100) / 2
```

- **返工率** = 被 Reopen 的任务数 / 已完成任务数
- **Bug 引入率** = 你完成的任务中关联了 Bug 的比例

### 💡 如何提升
- **一次做对**：减少被 Reopen 的次数
  - 开发前理解清楚需求
  - 自测充分再提交
  - Code Review 前自查
- **减少 Bug**：
  - 写单元测试
  - 关注边界情况
  - 做好 QA 自测
- **返工率为 0 + Bug 引入率为 0 = 质量满分 100**

---

## 4. 影响力（15%）— 你做的事有多重要

### 怎么算的
```
影响力 = 高优完成比×100×60% + 阻塞解决速度×40%
```

- **高优完成比**：完成的 P0/P1 任务占你所有完成任务的比例
- **阻塞解决速度**：有关联 issue 的任务完成得多快

### 💡 如何提升
- **主动接高优先级任务**：P0/P1 的任务权重占 60%
- **快速解决阻塞性问题**：如果你的任务阻塞了别人，尽快完成
- **不要只挑简单低优任务**：P3 任务对影响力没贡献
- **帮团队解决卡点**：blocked 的任务你去推动，对影响力加分

---

## 5. 协作（15%）— 你帮助团队多少

### 怎么算的
```
协作 = (评论分量 + 任务参与度) / 2
评论分量 = min(100, 跨团队评论数 × 10)
任务参与度 = 你参与评论的他人任务数 / 他人任务总数 × 100
```

- **跨团队评论**：在不是你负责的任务上发表评论
- **任务参与度**：你帮助了多少别人的任务

### 💡 如何提升
- **积极参与 Code Review**：在别人的 ticket 上留 comment
- **帮助同事解决问题**：在他人任务上提供建议
- **10 条跨团队评论即可满分**：不需要很多，关键是参与
- **参与更多人的任务**：覆盖面广比深度更重要
- **不要只关注自己的任务**：团队协作是加分项

## 快速提分行动清单

| 优先级 | 行动 | 影响维度 | 预期提升 |
|--------|------|----------|----------|
| ⭐⭐⭐ | 及时更新 Jira 状态（In Progress/Done） | 效率 | +20~30 分 |
| ⭐⭐⭐ | Sprint 内完成承诺的任务 | 效率+吞吐量 | +15~25 分 |

| ⭐⭐ | 主动接 P0/P1 任务 | 影响力 | +20~40 分 |
| ⭐⭐ | 接复杂任务（多子任务/多关联） | 吞吐量 | +10~20 分 |
| ⭐ | 拆分大任务，缩短 Cycle Time | 效率 | +10~15 分 |

## 常见问题

**Q: 为什么我完成了很多任务但吞吐量分不高？**
A: 吞吐量是团队内相对排名。如果别人完成得更多，你的百分位就低。另外简单任务复杂度因子只有 1.0，大任务可以到 5.0。

**Q: 我的效率分为什么是 0？**
A: 检查 Jira 状态更新。如果你没有把任务移到 In Progress 就直接 Done，系统无法计算 Cycle Time，会用 createdAt → updatedAt 作为 fallback（通常很长）。

**Q: 怎样让质量满分？**
A: 不被 Reopen + 你完成的任务没有关联 Bug = 100 分。

**Q: 协作分怎么快速提升？**
A: 最简单的方式：在 10 个不同同事的 Jira ticket 上各留一条有价值的评论。

**Q: 为什么影响力分很低？**
A: 如果你只做 P2/P3 任务，高优完成比为 0，影响力最多只有 40 分（阻塞分量的中性值 50 × 40% = 20）。主动接 P0/P1 任务是关键。
