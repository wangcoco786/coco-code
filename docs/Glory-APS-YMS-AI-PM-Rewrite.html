Reseller 申请核心流程（两种场景）

👤 场景1：游客申请（无 IAM 账号）

  • 1. 游客在 Chatbot 填写申请表单（邮箱+密码+公司信息+上级 Reseller）
  • 2. 前置校验：邮箱是否已存在 / 已是 Reseller
  • 3. 校验通过 → 创建申请记录（PENDING_APPROVAL）
  • 4. 审批通过 → 状态变更 APPROVING
  • 5. Marketplace 调用 BNP 创建 Company + Reseller（含产品基本信息）
  • 6. Marketplace 调用 IAM 创建 C 端用户账号
  • 7. 创建站点草稿 → 状态 APPROVED → 通知用户可登录

🔑 场景2：已登录用户申请（已有 IAM 账号）

  • 1. 已登录用户填写申请（公司信息+上级 Reseller，无需邮箱密码）
  • 2. 从 Token 获取 userId，创建申请记录（PENDING_APPROVAL）
  • 3. 审批通过 → 状态变更 APPROVING
  • 4. Marketplace 调用 BNP 创建 Company + Reseller（传入已有 userId）
  • 5. 创建站点草稿 → 状态 APPROVED
  • 6. 通知用户：当前账号已关联 Reseller 身份
  • 无需创建 IAM 账号，使用已有账号身份叠加

核心目标：实现仓库司机自动化 Check-in，减少人力 · 硬件外采 + 软件自研
🎯 核心目标：通过部署边缘工控机（YMS-Box）+ 摄像头，实现物流园区司机自动化 Check-in，取代人工登记流程，显著减少现场人力投入。
技术方案：采用 MQTT 协议（EMQX Broker）实现指令交互 + TCP/IP 双向通信实现视频流传输，解决原有 HTTP 方案的域名爆炸问题。

🔧 技术方案要点

  • MQTT 指令交互：YMS 服务端通过 EMQX Broker 与边缘工控机双向通信，下发开启视频流等指令
  • 设备状态监控：EMQX 实时获取工控机在线/离线状态，离线自动告警触发现场排查
  • TCP/IP 视频流：摄像头实时视频采用 TCP/IP 双向通信，边缘设备为客户端，云端为服务端，无需申请域名和固定IP
  • 手机端验证：支持手机APP查看摄像头监控视频，确认设备部署就绪

📋 实施组成

  • 硬件（外采）：工控机、摄像头、道闸设备、显示屏
  • 软件（自研）：YMS-Box 客户端（Msg/Video/Cmd/Local Service）、消息中台、视频中台
  • 工控激活流程：厂家申请硬串号 → YMS 生成待激活硬串号 → 打印贴码 → 实施端 App 扫码绑定激活
