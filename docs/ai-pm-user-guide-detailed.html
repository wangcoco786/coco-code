9. 部门绩效（详细说明）

9.1 综合绩效公式

绩效分 = 吞吐量×20% + 效率×25% + 质量×25% + 影响力×15% + 协作×15%

9.2 绩效等级

🟢 80-100 Excellent（优秀） 🔵 60-79 Good（良好） 🟠 40-59 Average（一般） 🔴 0-39 Needs Improvement（需改进）

9.3 优先级说明（影响绩效计算）

⚠️ 重要：Jira 任务的优先级直接影响「影响力」维度得分。高优先级任务完成越多，影响力分越高。
Jira 优先级平台映射对绩效的影响建议
HighestP0高优完成比 +1（影响力核心指标）⭐⭐⭐ 优先完成，直接提升影响力分
HighP1高优完成比 +1（影响力核心指标）⭐⭐⭐ 优先完成，直接提升影响力分
MediumP2不计入高优完成比⭐⭐ 正常处理，贡献吞吐量
Low / LowestP3不计入高优完成比⭐ 有空再做
💡 关键点：影响力维度中，高优先级完成比 = 你完成的 P0+P1 任务数 / 你完成的总任务数。如果你只做 P2/P3 任务，这个比值为 0，影响力最多只能得 20 分（阻塞分量的中性值）。

9.4 五维度详细说明

① 吞吐量（权重 20%）— 产出能力

统计 Sprint 内完成的任务数 × 复杂度因子，在团队中算百分位排名。

复杂度因子条件加分
子任务 >3 个+0.5
子任务 >6 个+1.0（与上互斥取高）
关联 issue >2+0.3
关联 issue >5+0.6（与上互斥取高）
评论 >5 条+0.3
评论 >10 条+0.6（与上互斥取高）
跨 Sprint 次数 >0+0.3 × 次数
💡 提升方法：多完成任务 + 接复杂任务（多子任务/多关联）。只有 status=Done 的任务才计入。

② 效率（权重 25%）— 交付速度

Cycle Time 分量 = max(0, 100 - 平均天数×5)。Delivery Rate 分量 = Sprint 结束前完成数/总分配数×100。两者各占 50%。

⚠️ 注意：如果不及时更新 Jira 状态（不移到 In Progress），系统无法准确计算 Cycle Time，会导致效率分偏低。
💡 提升方法：开始做任务就移到 In Progress，做完立刻移到 Done。拆分大任务（20 天 Cycle Time = 0 分）。Sprint 内完成承诺的任务。

③ 质量（权重 25%）— 交付质量

返工率 = 被 Reopen 次数 / 已完成数。Bug 引入率 = 关联 Bug>0 的完成任务 / 已完成总数。

质量 = ((1-返工率)×100 + (1-Bug引入率)×100) / 2
💡 提升方法：一次做对（减少 Reopen）+ 减少 Bug（自测充分、写单元测试）。返工率 0 + Bug 引入率 0 = 满分 100。

④ 影响力（权重 15%）— 高价值贡献

影响力 = 高优完成比×100×60% + 阻塞解决速度分量×40%

高优 = P0/P1（Highest/High）。阻塞解决速度分量 = max(0, 100 - 阻塞任务平均 Cycle Time × 5)。无阻塞任务时取中性值 50。

💡 提升方法：主动接 P0/P1 任务 + 快速完成有 blocked 关系的任务。

⑤ 协作（权重 15%）— 团队贡献

协作 = (min(100, 跨团队评论数×10) + 他人任务参与比×100) / 2

跨团队评论 = 在不是你负责的任务上发表的评论数。10 条即满分。

💡 提升方法（最容易提分）：在 10 个不同同事的 Jira ticket 上各留一条有价值的评论（Code Review、建议、问题讨论）。

9.5 快速提分优先级

优先级行动影响维度预期提升
⭐⭐⭐及时更新 Jira 状态（In Progress → Done）效率+20~30
⭐⭐⭐Sprint 内完成承诺的任务效率+吞吐量+15~25
⭐⭐⭐减少 Reopen（一次做对）质量+10~25
⭐⭐在他人任务上留 10 条评论协作+50（满分）
⭐⭐主动接 P0/P1 任务影响力+20~40
⭐⭐接复杂任务（多子任务/多关联）吞吐量+10~20
⭐拆分大任务，缩短 Cycle Time 到 4 天以内效率+10~15
