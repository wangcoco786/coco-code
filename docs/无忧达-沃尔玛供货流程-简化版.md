# 无忧达 × 沃尔玛供货流程（概念精简版）

## 目标

帮助无忧达建立统一的北美海外仓面对大卖场（Walmart）的供货流程。

## 业务链路

无忧达卖家 → 无忧达海外仓（北美）→ Walmart DC（配送中心）

## Walmart 合规要求

### EDI 报文（X12 标准）

| EDI 报文 | 方向 | 用途 | 时效要求 |
|---|---|---|---|
| 850 (PO) | Walmart → 供应商 | 采购订单 | 收到后次日 4pm CST 前确认 |
| 855 (PO Ack) | 供应商 → Walmart | 订单确认 | 收到 PO 后 24h 内 |
| 856 (ASN) | 供应商 → Walmart | 发货通知 | 发货当天，货到 DC 前 |
| 810 (Invoice) | 供应商 → Walmart | 发票 | 发货后 |
| 997 (FA) | 双向 | 功能确认 | 自动 |

### GS1-128 标签

每个出库箱/托盘必须贴合规标签，包含 SSCC-18（全球唯一序列货运容器代码）、GS1-128 条码编码 SSCC。每个 SSCC 必须在 856 ASN 中出现，否则 DC 拒收。

### Routing Guide

- **MABD**：Must Arrive By Date，PO 上指定的到货截止日
- 发往 Walmart DC 需提前预约送货窗口
- 托盘规格：GMA 标准托盘 48×40 英寸，最高 60 英寸

### OTIF 考核

- OTIF（On-Time In-Full）准时足量率要求 ≥98%，低于扣 3% 货款
- On-Time：货物在 MABD 当天或之前到达
- In-Full：PO 中每个 line item 数量 100% 齐全

## 系统能力现状

| 状态 | 内容 |
|---|---|
| ✅ 已具备 | EDI 解析 X12；自动生成 855；实时库存查询；波次拣货；出库自动生成 856 ASN；可生成 810 |
| ⚠️ 需完善 | 标签模板合规确认；配置 GS1 Company Prefix；新增 MABD 倒排逻辑 |
| ❌ 缺失 | OTIF 看板 |

## 四大系统职责

| 系统 | 角色 |
|---|---|
| **EDI** | 商超的"数字握手通道"，自动收发 PO、ASN、Invoice |
| **DI** | 数据翻译官，不同格式互转（X12/JSON/XML），不同商超统一适配 |
| **OMS** | 业务大脑，接单→分仓→排程→监控，订单全生命周期管理 |
| **WMS** | 仓内执行引擎，收货、上架、拣货、包装、贴标、出库 |

## 必须落地的能力（合规性）

1. Walmart EDI 850/855/856/810 全流程合规
2. GS1-128 标签打印（含 SSCC-18）
3. SSCC 与 ASN 严格一致
4. PO 确认时效自动化（24h）

## 强烈建议做（避免罚款）

5. MABD 倒排排程（从 MABD 反推最晚出库日）
6. OTIF 达成率看板
7. Routing Guide 规则引擎（自动匹配承运商、预约 DC 窗口）
8. 短装/超装校验

## 主要风险

| 风险 | 影响 |
|---|---|
| SSCC 不一致 | DC 拒收，退货 + 扣罚 |
| ASN 晚于货物到达 | OTIF 不达标 |
| 标签不合规 | 扣罚 + 退货 |
| MABD 管控缺失 | 大面积迟到 |
| GS1 Company Prefix 未注册 | 无法生成合规 SSCC |

## 实施节奏

- **Phase 1（MVP，2-4周）**：全流程打通 + GS1-128 标签合规
- **Phase 2（合规加固，2-4周）**：SSCC 与 ASN 一致性校验 + OTIF 监控看板
- **Phase 3（规模化，持续）**：多卖家多仓多 DC 路由 + Routing Guide 规则引擎 + Chargeback 分析
